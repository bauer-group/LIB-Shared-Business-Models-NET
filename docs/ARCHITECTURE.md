# Architecture

This repository hosts the BAUER GROUP **POCO domain-model** layer: plain data
types with no behavior beyond construction defaults. It is intentionally thin —
the persistence, transport, validation and service layers live elsewhere and
consume these models.

The DAG is enforced by NetArchTest architecture tests
(`tests/BAUERGROUP.Shared.Business.Models.UnitTests/Architecture`): the base
package must not depend on any satellite or on Core, satellites must not depend
sideways onto each other.

## Dependency policy

- The **base** `BAUERGROUP.Shared.Business.Models` package ships **zero** runtime
  NuGet dependencies — pure .NET BCL surface.
- **Domain satellites** may layer on `BAUERGROUP.Shared.Core` (`[3.0.0,4.0.0)`)
  for shared primitives (e.g. `Core.Images.IndependentImage`,
  `Core.Extensions` helpers). Nothing else is permitted.

## Base types

| Type | Kind | Purpose |
| --- | --- | --- |
| `IBusiness` | interface | Marker for any business type. |
| `IBusinessObject` | interface | `IBusiness` + `Guid UID` (init-only) and `DateTime Changed`. |
| `Business` | abstract class | Base for types without identity. |
| `BusinessObject` | abstract class | Base with auto-generated `UID` + UTC `Changed`. |

### Identity (`UID`) and timestamp (`Changed`)

`UID` is `init`-only — assignable at construction (object initializer **or**
the `BusinessObject(Guid gUID)` constructor) and immutable afterwards. A fresh
`Guid` is generated by default; passing `Guid.Empty` falls back to that
auto-generation. `Changed` is a mutable UTC timestamp the persistence layer
stamps before each save.

```csharp
new Customer()                       // UID = new Guid, Changed = UtcNow
new Customer(knownId)                // UID = knownId   (re-hydration)
new Customer { UID = knownId }       // UID = knownId   (object initializer)
```

## Cloning

Deep copies use the `BAUERGROUP.Shared.Core` JSON helper
(`Core.Extensions.ObjectHelper.Clone<T>`), a safe `System.Text.Json`
serialize/deserialize round-trip — **not** the removed `BinaryFormatter`. The
trade-off vs. `record` `with`-expressions: `with` is a *shallow* copy, whereas
the JSON helper is a true *deep* copy. The models stay plain classes (reference
equality preserved — backward-compatible); value-semantics were intentionally
**not** adopted. See [MIGRATION.md](MIGRATION.md) for the round-trip constraint
on some legacy ERP types.

## Conventions

- **Namespaces** mirror the package id (`BAUERGROUP.Shared.Business.Models.<Domain>`).
  Because the satellite namespaces nest under the base namespace, base types
  resolve without an explicit `using`.
- **Hungarian notation** is the house style for constructor parameters and
  locals (`sName`, `eType`, `gUID`); public property names are clean PascalCase.
  `CA1720`/`CA1716` are suppressed repo-wide to accommodate this (see
  `Directory.Build.props`).
- **Analyzer strategy:** hand-written packages (base, CRM, DMS) build under the
  strict bar (`Nullable=enable`, `AnalysisMode=AllEnabledByDefault`,
  warnings-as-errors). Migrated legacy packages (Shipping, ERP) relax to
  `Nullable=disable` / `AnalysisLevel=latest-default` / non-fatal warnings so the
  legacy surface compiles verbatim; hardening is a deliberate later pass — the
  nullable-enable changeover is fully planned in
  [NULLABLE-ENABLE.md](NULLABLE-ENABLE.md).
